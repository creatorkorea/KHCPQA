# QA and Release

## 검수 기준

| 항목 | 합격 기준 |
| --- | --- |
| 반응형 | 모바일 375px, 태블릿 768px, 데스크톱에서 주요 화면을 읽고 버튼 사용 가능 |
| 페이지 | 계약 범위의 주요 라우트가 존재하고 404 없이 접근 가능 |
| 콘텐츠 | 발주사가 제공하거나 기존 사이트에서 가져온 최종 문구와 이미지 반영 |
| 회원/자격 조회 | 테스트 계정과 샘플 자격 데이터 기준으로 기본 흐름 작동 |
| 관리자 | 관리자 전용 접근, 주요 콘텐츠 등록/수정/목록 확인 가능 |
| 다국어 | 한국어/영어/스페인어 라우트와 언어 선택기 작동 |
| 보안 | 관리자와 My Page는 비로그인 접근 차단 |

## 필수 테스트

- 홈 렌더링
- 공개 메뉴 이동
- 언어 전환
- 과정 목록/상세
- 활동 목록/상세
- 문의 폼 제출
- 회원가입
- 로그인
- 로그아웃
- My Page 접근 제어
- 자격 조회
- 관리자 로그인
- 관리자 콘텐츠 등록/수정
- 관리자 문의 확인
- 관리자 자격 데이터 등록/수정

## 반응형 테스트

필수 viewport:
- 375px
- 768px
- 1280px 이상

확인 항목:
- 헤더 깨짐 없음
- 메뉴 터치 가능
- 카드 텍스트 넘침 없음
- 버튼 텍스트 잘림 없음
- 이미지 비율 깨짐 없음
- 폼 입력 가능
- 관리자 표가 모바일에서 사용 가능하거나 데스크톱 우선 안내 제공

## SEO 테스트

- 공개 페이지 title 확인
- description 확인
- Open Graph 확인
- canonical 확인
- hreflang 확인
- sitemap.xml 확인
- robots.txt 확인
- 로그인 후 페이지 noindex 확인

## 검수 프로세스

1. 개발자가 검수 URL을 제공한다.
2. 발주사는 7영업일 이내 승인 또는 구체적 수정 요청을 전달한다.
3. 미응답 시 재요청한다.
4. 재요청 후 3영업일 내 응답이 없으면 조건부 승인으로 본다.
5. 수정 요청은 계약 범위 내 오류, 누락, 오탈자, 링크 오류, 반응형 표시 문제, 문구/이미지 교체에 한정한다.

## 릴리즈 전 금지 사항

- 실제 개인정보를 테스트 환경에 노출 금지
- 검수되지 않은 영어/스페인어를 최종 공개 금지
- 관리자 계정 정보를 평문 공유 금지
- 공개 페이지에서 자격 상세 개인정보 노출 금지
