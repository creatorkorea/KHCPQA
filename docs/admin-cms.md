# Admin CMS

관리자 페이지는 비개발자가 콘텐츠와 자격 데이터를 운영할 수 있도록 단순하고 명확해야 한다.

## 관리자 메뉴

- 대시보드
- 페이지 관리자
- 과정 관리자
- 커뮤니티 관리자
- 후기 관리자
- 갤러리/미디어 관리자
- 문의 관리자
- 사용자 관리자
- 자격/수료증 데이터 관리자
- 팝업/배너 관리자
- 번역 상태 관리자

## 권한

| 역할 | 권한 |
| --- | --- |
| Super Admin | 전체 관리 |
| Content Manager | 페이지, 게시글, 갤러리 관리 |
| Course Manager | 과정 콘텐츠 관리 |
| Certification Manager | 자격 데이터 관리 |
| Inquiry Manager | 문의 확인 및 상태 변경 |
| Viewer | 읽기 전용 |

## 콘텐츠 상태

- draft
- translated
- reviewed
- published
- archived

## 과정 관리자 입력 규칙

과정 관리 탭은 `admin_content_items`의 `Course` 타입을 사용한다.

- `main`: 과정 목록 카드와 상세 상단 제목/요약/본문을 교체한다.
- `flow-1`~`flow-4`: 상세 페이지 커리큘럼 흐름 카드에 노출한다.
- `panel-goal`, `panel-strength`, `panel-audience`: 상세 페이지 핵심 안내 패널에 노출한다.
- `technique-1`: 상세 페이지 기술 키워드 영역에 노출한다.
- `process-1`: 상세 페이지 실습 프로세스 영역에 노출한다.

본문은 줄바꿈 목록으로 입력하면 공개 상세 페이지에서 항목형 콘텐츠로 사용된다.

## 관리자 화면 요구사항

- 목록, 검색, 필터, 정렬 제공
- 언어별 콘텐츠 입력 필드 제공
- 게시 상태 변경 제공
- 대표 이미지 업로드와 미리보기 제공
- 대표 이미지 URL 입력 제공
- 원본 URL 저장 필드 제공
- 최근 수정자와 수정일 표시
- 삭제보다 비공개/보관 처리 우선

현재 구현 기준:
- Supabase 환경변수가 설정된 운영/검수 환경에서는 DB 조회 결과가 0건이면 빈 상태를 표시한다.
- Supabase 환경변수가 없는 로컬 프리뷰에서만 정적 데모 행을 사용한다.
- 저장/삭제 액션은 Supabase 환경변수가 없으면 성공으로 처리하지 않고 설정 오류를 표시한다.
- 콘텐츠와 배너는 `image_url`을 저장할 수 있으며, 공개 과정/활동 화면은 published CMS 이미지 URL을 우선 사용한다.

## 문의 관리

문의 유형:
- `general`
- `course`
- `certification`
- `partnership`

필드:
- 이름
- 기관명
- 국가
- 이메일
- 연락처
- 관심 분야
- 메시지
- 개인정보 동의 여부
- 처리 상태: `new`, `in_review`, `answered`, `closed`
- 담당자 메모

문자/카카오/이메일 자동 발송은 1차 범위가 아니다.

## 자격 데이터 관리

필드:
- 사용자
- 과정
- 자격증 번호
- 발급일
- 상태: `issued`, `expired`, `revoked`
- 검증 코드
- 관리자 메모

주의:
- 실제 자격 데이터의 정확성은 발주사 책임이다.
- 공개 검증 기능은 별도 확정 전까지 기본 범위로 보지 않는다.
- 증명서 자동 발급은 제외 범위다.

## 관리자 보안

- 관리자 라우트는 인증 필수
- 권한별 접근 제한
- 공개 가입 사용자는 기본 `user` 권한으로 생성하며 관리자 권한은 별도 부여
- 비밀번호 정책 적용
- 관리자 페이지 `noindex`
- 자격 데이터 다운로드 기능은 별도 승인 후 구현
