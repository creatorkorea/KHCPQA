# Admin CMS

관리자 페이지는 비개발자가 콘텐츠와 자격 데이터를 운영할 수 있도록 단순하고 명확해야 한다.

## 관리자 메뉴

- 대시보드
- 페이지 관리자
- 과정 관리자
- 커뮤니티 관리자
- 후기 관리자
- 갤러리/미디어 관리자
- 문의 관리자
- 사용자 관리자
- 자격/수료증 데이터 관리자
- 팝업/배너 관리자
- 번역 상태 관리자

## 권한

| 역할 | 권한 |
| --- | --- |
| Super Admin | 전체 관리 |
| Content Manager | 페이지, 게시글, 갤러리 관리 |
| Course Manager | 과정 콘텐츠 관리 |
| Certification Manager | 자격 데이터 관리 |
| Inquiry Manager | 문의 확인 및 상태 변경 |
| Viewer | 읽기 전용 |

## 콘텐츠 상태

- draft
- translated
- reviewed
- published
- archived

## 관리자 화면 요구사항

- 목록, 검색, 필터, 정렬 제공
- 언어별 콘텐츠 입력 필드 제공
- 게시 상태 변경 제공
- 대표 이미지 업로드와 미리보기 제공
- 원본 URL 저장 필드 제공
- 최근 수정자와 수정일 표시
- 삭제보다 비공개/보관 처리 우선

## 문의 관리

문의 유형:
- `general`
- `course`
- `certification`
- `partnership`

필드:
- 이름
- 기관명
- 국가
- 이메일
- 연락처
- 관심 분야
- 메시지
- 개인정보 동의 여부
- 처리 상태: `new`, `in_review`, `answered`, `closed`
- 담당자 메모

문자/카카오/이메일 자동 발송은 1차 범위가 아니다.

## 자격 데이터 관리

필드:
- 사용자
- 과정
- 자격증 번호
- 발급일
- 상태: `issued`, `expired`, `revoked`
- 검증 코드
- 관리자 메모

주의:
- 실제 자격 데이터의 정확성은 발주사 책임이다.
- 공개 검증 기능은 별도 확정 전까지 기본 범위로 보지 않는다.
- 증명서 자동 발급은 제외 범위다.

## 관리자 보안

- 관리자 라우트는 인증 필수
- 권한별 접근 제한
- 공개 가입 사용자는 기본 `user` 권한으로 생성하며 관리자 권한은 별도 부여
- 비밀번호 정책 적용
- 관리자 페이지 `noindex`
- 자격 데이터 다운로드 기능은 별도 승인 후 구현
