# Client Review Request

이 문서는 발주사 검수 URL 전달 전에 확정이 필요한 항목과 검수 요청 범위를 정리한다.

## 검수 전 확정 필요

- [ ] 최종 브랜드 표기: `KHCPQA`, `한국건강관리사자격협회`, `SMC아카데미` 병기 방식
- [ ] 계약 범위와 제외 범위 최종 확인
- [ ] 도메인, 호스팅, SSL 비용과 계정 명의
- [ ] 기존 SMC365 이미지, 후기, 보도자료, 강사 사진 사용 권리
- [ ] 개인정보처리방침 원문
- [ ] 이용약관 원문
- [ ] 실제 자격 데이터 샘플 CSV 또는 엑셀
- [ ] 초기 관리자 계정 수, 역할, 비밀번호 정책
- [ ] 영어/스페인어 최종 감수 담당자

## 검수 요청 범위

- `/ko`, `/en`, `/es` 공개 페이지 구조
- 협회 소개, 과정 목록/상세, 활동/커뮤니티, Contact
- 파트너 문의 제출과 관리자 문의 확인
- 회원가입, 로그인, My Page, Profile, Certification Inquiry
- 관리자 콘텐츠, 문의, 자격 데이터, 배너 관리
- 모바일 375px, 태블릿 768px, 데스크톱 화면 표시
- SEO, `sitemap.xml`, `robots.txt`, 비공개 페이지 `noindex`

## 검수 완료 기준

- 주요 라우트 404 없음
- 공개 메뉴에 My Page 없음
- 로그인 후 계정 메뉴와 My Page 접근 가능
- 문의 제출 데이터가 관리자 문의함에 저장됨
- 자격 샘플 데이터가 로그인 회원에게 정상 표시됨
- 관리자 비로그인 접근이 차단됨
- 발주사 제공 최종 문구와 이미지가 반영됨

## 현재 QA 결과

- `npm run lint` 통과
- `npm run build` 통과
- 원격 Supabase 마이그레이션 적용 확인
- 원격 Supabase RLS 정책 확인
- 문의 제출 E2E 통과
- 로그인/계정 접근/자격 조회 E2E 통과
- 반응형 QA 통과

상세 기록은 `docs/qa-release.md`를 기준 문서로 사용한다.
