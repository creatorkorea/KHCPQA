# Agent Development Guide

이 문서는 KHCPQA 글로벌 웹앱을 개발하는 에이전트와 개발자가 가장 먼저 읽어야 하는 작업 지침이다.

## 1. 기준 문서

개발 판단 우선순위:

1. `260704_홈페이지 제작 계약서.pdf`
2. `KHCPQA_WEBAPP_DEVELOPMENT_PLAN.md`
3. `6_23_KHCPQA_Global_Website_App_Blueprint_KO_Revised_EN_ES.docx`
4. 기존 콘텐츠 출처: `https://www.smc365.ac/index.asp`

계약서에 포함되지 않은 기능은 구현하지 않는다. 필요하면 별도 견적 또는 후속 단계로 분리한다.

## 2. 반드시 지킬 원칙

- My Page는 공개 상단 메뉴와 공개 사이트맵에 노출하지 않는다.
- 로그인 후 계정 메뉴에서만 My Page, Profile, Certification Inquiry, Logout을 표시한다.
- 한국어, 영어, 스페인어 3개 국어 라우팅을 기본으로 설계한다.
- 기존 SMC365 콘텐츠를 원천으로 사용하되, 신규 사이트는 글로벌 협회형 톤으로 재구성한다.
- 관리자와 My Page는 비로그인 접근을 차단한다.
- 로그인 후 페이지는 `noindex` 처리한다.
- 자격 데이터와 개인정보는 최소 노출한다.
- 결제, 알림, 네이티브 앱, 자동 번역, 증명서 자동 발급, 고급 통계, 대량 마이그레이션은 1차 범위에서 제외한다.

## 3. 권장 개발 순서

1. 라우팅, i18n, 레이아웃 구조를 먼저 잡는다.
2. 디자인 토큰과 공통 컴포넌트를 만든다.
3. 공개 페이지 템플릿을 만든다.
4. 과정/활동 콘텐츠 모델을 만든다.
5. 로그인, 계정 메뉴, My Page 접근 제어를 만든다.
6. 자격 조회 샘플 플로우를 만든다.
7. 관리자 CRUD를 만든다.
8. 기존 사이트 콘텐츠를 우선순위에 따라 이관한다.
9. 반응형, 다국어, 권한, SEO QA를 수행한다.

## 5. 개발 중 판단 규칙

- 애매하면 계약 범위를 좁게 해석한다.
- 새 기능을 추가하기보다 관리자에서 운영 가능한 구조를 우선한다.
- 기존 사이트 구조를 그대로 복제하지 말고, 신규 글로벌 IA에 맞춰 재배치한다.
- 콘텐츠가 부족한 언어는 임시 자동 번역처럼 보이게 공개하지 않는다.
- 공개 페이지는 SEO와 신뢰도를 우선하고, 관리자 페이지는 반복 운영 효율을 우선한다.

## 6. 완료 정의

1차 개발 완료는 다음 조건을 모두 만족해야 한다.

- `/ko`, `/en`, `/es` 주요 공개 라우트가 작동한다.
- 공개 메뉴에 My Page가 없다.
- 로그인 후 계정 메뉴와 My Page가 작동한다.
- 과정 목록과 주요 과정 상세 템플릿이 작동한다.
- 활동/커뮤니티 목록과 상세 템플릿이 작동한다.
- 문의 폼이 관리자 문의함에 저장된다.
- 자격 조회 샘플 데이터가 표시된다.
- 관리자에서 주요 콘텐츠를 등록/수정/조회할 수 있다.
- 모바일 375px, 태블릿 768px, 데스크톱에서 핵심 화면이 깨지지 않는다.
